Definitions And Options

This page is the plain-English companion to the scenario TOML files. It groups parameters by purpose, explains what each one is for, and calls out the most important option values without forcing the config files themselves to become a handbook.

Scenario And Display

High-level metadata that names the scenario and controls how it appears in the selector and chart chrome.

[scenario].name

Human-friendly scenario name shown in the shell and editor.

[scenario].slug

Stable file-safe id used in URLs, output folders, and editor routing.

[scenario].description

Short plain-English description of what is different about the scenario.

[scenario].is_default

Marks the scenario the public selector should open first.

  • true = default starting scenario
  • false = alternate scenario
[display].projection_title

Subtitle text shown in the projection page header/chart title area.

Data Source And Synthetic Start

Controls where year-0 balances come from. Most real household scenarios use Monarch data; demo/share-safe scenarios use synthetic balances.

[data_source].mode

Choose whether the scenario starts from live/cached Monarch balances or explicit synthetic amounts.

  • monarch = live or cached classified account balances
  • synthetic = balances come from [synthetic_start]
[synthetic_start].taxable / trad_ira / roth / cash

Starting investable balances for synthetic scenarios.

[synthetic_start].home_value / vehicles / other

Starting non-investable rollups for synthetic scenarios.

[synthetic_start.property_values]

Optional named real-estate balances so future `SellHome` events can target a specific property.

[synthetic_start.liability_balances]

Optional named starting liability balances keyed to `[[liabilities]].name`.

[synthetic_start.retirement_owner_balances]

Optional owner split for starting traditional/Roth balances when one combined number is not good enough.

[synthetic_start].taxable_cost_basis

Optional explicit starting cost basis inside the synthetic taxable account.

[synthetic_start].roth_contribution_basis

Optional explicit starting Roth contribution basis, separate from Roth earnings.

Simulation

Controls the time range and which engine mode is rendered.

[simulation].start_year / end_year

Projection window used by the yearly model.

[simulation].clamp_start_year

When true and balance data has a known as-of date newer than start_year, the projection start is auto-adjusted forward to match the data year. Set to false to disable.

  • true (default)
  • false
[simulation].real_dollar_basis

When true, all chart and table figures are shown in start-year purchasing power (deflated by cumulative inflation). False (default) shows nominal future-year values.

  • false (default) — nominal dollars
  • true — deflated to start-year dollars
[simulation].render_modes

Which pre-rendered modes the scenario shell should build and expose.

  • deterministic
  • historical
  • monte_carlo
[simulation].mode

Single-run compatibility setting; the renderer overrides this per mode when batch rendering multiple outputs.

  • deterministic = one straight-line run
  • historical = rolling historical return windows
  • monte_carlo = randomized annual return paths
[simulation].num_runs

Number of Monte Carlo runs to simulate.

[simulation].seed

Optional random seed so Monte Carlo results are repeatable.

[simulation].portfolio_return_volatility

Annual return variability used by Monte Carlo around the blended stock/bond return.

[simulation].historical_returns_path

CSV file with `year,return` columns used to create rolling historical return sequences.

Stochastic Success Rules

Used by both Monte Carlo and historical modes to decide whether a run counts as a success or failure.

[monte_carlo.success].failure_mode

Main failure rule for stochastic reporting.

  • net_worth_below_zero = fail once total net worth drops below zero
  • liquid_depletion = fail when investable liquid assets are exhausted
  • spending_shortfall = fail when required spending is not fully funded
  • preserve_home_equity = fail when the plan needs to consume protected home equity
  • custom = compare one output column against a threshold
[monte_carlo.success].minimum_spending_funded_ratio

Funding threshold used by `spending_shortfall` mode. `1.0` means fully funded.

[monte_carlo.success].allow_home_equity_for_spending

Whether home equity can count as available funding in success calculations.

[monte_carlo.success].allow_debt_for_spending

Whether going negative / borrowing can still count as funded in success calculations.

[monte_carlo.success].failure_grace_period_months

Lets a run experience temporary pressure before it becomes a true failure.

[monte_carlo.success].custom_failure_column / operator / threshold

Advanced custom comparator used only when `failure_mode = "custom"`.

People

Each household member gets an independent profile. `person1` and `person2` are the canonical keys used across the app.

[personX].name

Display name used in tables, legends, and synthesized event labels.

[personX].dob

Date of birth used for age labels, end-of-plan timing, and RMD age checks.

[personX].life_expectancy

Modeled terminal age, not a prediction.

[personX].retirement_year

First model year in which wage income stops for that person.

[personX].annual_take_home

Annual wage cashflow input. Usually treated as net cash unless taxes are switched to `taxable_wages` mode.

[personX].annual_take_home_is_net_of_retirement_contributions

Use `true` when 401(k)/similar payroll contributions are already excluded from take-home pay.

[personX].annual_take_home_real_raise

Real wage growth above inflation.

[personX].annual_401k_contribution

Total annual workplace retirement contribution input for that person.

[personX].annual_401k_contribution_extra_increase

Additional annual growth applied to the 401(k) contribution path beyond the wage-growth path.

[personX].contribution_method

401(k) contribution method: `flat` (default) or `percent_of_gross`. In `percent_of_gross` mode, contributions are computed from gross_income × retirement_contribution_percent.

[personX].gross_income

Gross annual income used for percentage-based 401(k) contribution computation. Only used when `contribution_method = "percent_of_gross"`.

[personX].gross_income_annual_increase_percent

Combined annual gross income increase (COLA + performance + merit) as a decimal. Example: 0.05 = 5% annual increase.

[personX].retirement_contribution_percent

401(k) contribution as a percentage of gross income. Example: 0.12 = 12% of gross.

[personX].retirement_contribution_annual_increase_percent

Annual percentage-point increase in 401(k) contribution rate (auto-escalation). Example: 0.02 = increase contribution rate by 2 percentage points each year.

[personX].retirement_contribution_max_percent

Maximum contribution percentage cap. The contribution rate stops escalating once it reaches this value. Example: 0.18 = never exceed 18% of gross income.

[personX].annual_401k_employer_match

Flat annual employer 401(k) match dollar amount. Used when `annual_401k_employer_match_mode = "flat"` (default).

[personX].annual_401k_employer_match_mode

Employer match mode: `flat` (default) or `percent_of_gross`. In percent mode, the employer matches `match_rate` of employee contributions up to `match_max_percent` of gross income.

[personX].annual_401k_employer_match_rate

Percentage of employee contribution the employer matches. Example: 0.50 = 50 cents per dollar contributed.

[personX].annual_401k_employer_match_max_percent

Maximum percentage of gross income the employer will match. Example: 0.06 = match applies only to the first 6% of salary contributed.

[personX].annual_ira_contribution

Direct IRA contribution modeled as a cash outflow plus deposit into `trad_ira` or `roth`.

[personX].annual_401k_contribution_bucket / annual_ira_contribution_bucket

Optional simple routing override for retirement contributions.

  • trad_ira
  • roth
[personX].annual_401k_contribution_split

Optional proportional split for one bundled workplace plan spanning traditional and Roth dollars.

[personX].rmd_trad_ira_share

Share of the household traditional balance attributed to that person for RMD math.

[personX].ss_start_age

Age when that person starts Social Security in the synthesized runtime config.

[personX].social_security_benefits

Age-to-monthly-benefit lookup used by the runtime Social Security event builder.

Spending

Baseline living-expense controls outside of one-off events.

[spending].retirement_annual

Household retirement spending target.

[spending].survivor_percent_of_retirement

Survivor spending target expressed as a share of the couple retirement target.

[spending].spending_basis

How to interpret the retirement/spending targets over time.

  • real = today’s dollars, then CPI-adjusted
  • nominal = fixed nominal amounts
[spending].pre_retirement_spending

Optional explicit pre-retirement annual spending target. Highest-precedence control.

[spending].annual_savings_override

Optional explicit pre-retirement savings target when you want to derive spending from savings rather than the default implied math.

Withdrawal Policy

Controls how NWN preserves cash reserves, covers deficits, and routes surplus cash in each life phase.

[withdrawal_policy].*_cash_target

Target cash reserve for `accumulation`, `retirement`, and `survivor` phases.

[withdrawal_policy].*_withdrawal_order

Bucket order used to cover deficits in each phase.

  • cash_above_target = only spend cash above the reserve floor
  • taxable = withdraw from taxable brokerage
  • trad_ira = withdraw from pre-tax retirement assets
  • roth = withdraw from Roth assets
  • cash_below_target = break the reserve itself only as a last resort
[withdrawal_policy].*_surplus_order

Where extra cash should be reinvested after the reserve is full.

  • taxable
  • roth
  • trad_ira

Taxes And RMD

Ordinary-income-focused household tax controls plus optional Required Minimum Distribution settings.

[taxes].enabled

Turns the bracket-based tax engine on or off.

[taxes].table_set

Shared tax reference dataset loaded from `config/tax_tables/`.

[taxes].*_filing_status

Federal/state filing status to use in `pre_retirement`, `retirement`, and `survivor` phases.

  • single
  • married_joint
  • head_of_household when supported by the table set
[taxes].wage_tax_treatment

How wage income should enter the tax model.

  • net_cash = wages stay as after-tax cashflow only
  • taxable_wages = wages are treated as taxable income
[taxes.rmd].enabled

Force RMD withdrawals from traditional retirement balances when age thresholds are met.

[taxes.rmd].start_age

Starting age for RMD enforcement.

[taxes.rmd.factors]

Optional age-to-divisor overrides for the lifetime table.

Market And Economic Assumptions

Return, inflation, and simplified fallback-tax assumptions used by the yearly engine.

[assumptions].stock_return / bond_return / equity_allocation

Defines the blended investable return path used in deterministic mode and as the center of stochastic mode.

[assumptions].inflation

CPI assumption used for real-dollar spending growth and some other inflation-linked paths.

[assumptions].cash_return

Separate growth rate for cash balances so reserves do not inherit the stock/bond blend.

[assumptions].real_estate_appreciation

Home/property appreciation assumption, independent from CPI.

[assumptions].real_estate_sale_fee_rate

Default transaction-cost assumption for `SellHome` events.

[assumptions].effective_tax_rate_pre_retirement / effective_tax_rate_post_retirement

Legacy fallback tax rates used only when the bracket-based tax engine is disabled/incomplete.

[assumptions].taxable_withdrawal_taxable_fraction

Legacy fallback only. If no explicit taxable basis seed is given, this implies the starting cost-basis share of the taxable account.

[assumptions].trad_ira_withdrawal_taxable_fraction

Taxable share of traditional-account withdrawals. Usually `1.0`.

[assumptions].initial_taxable_cost_basis_fraction

Optional direct opening assumption for how much of the taxable account is cost basis.

[assumptions].initial_roth_contribution_basis_fraction

Optional opening assumption for how much of the Roth balance is contribution basis rather than earnings.

Accounts Metadata

Maps live Monarch account names into NWN’s simplified model buckets and lets you add better opening metadata where needed.

[accounts]

Main account-classification block keyed by exact Monarch account name.

Category strings

Simple classification for one account.

  • taxable
  • trad_ira
  • roth
  • cash
  • real_estate
  • vehicle
  • liability
  • ignore
{ category = "...", owner = "..." }

Inline-table form used when you need owner attribution or extra metadata.

  • owner = "person1"
  • owner = "person2"
opening_balance_split

Splits one live account across multiple investable buckets at year 0.

basis_fraction

Optional taxable-account metadata: share of the starting taxable balance treated as cost basis.

roth_contribution_basis_fraction

Optional Roth-account metadata: share of the starting Roth balance treated as contribution basis.

[accounts].disabled

Exact account names to exclude without deleting the underlying category mapping.

Events

Discrete scenario changes. Each `[[events]]` item has `enabled`, `type`, `label`, and then type-specific fields.

Common event fields

Use `enabled = true/false` to toggle, keep `label` human-friendly, and prefer exact years over prose.

Retire

Stops a person’s wage income from its `year` onward.

  • Fields: person, year
SocialSecurity

Adds Social Security income for a person from its `year` onward.

  • Fields: person, year, monthly_benefit
  • Optional: taxable, taxable_fraction
Expense

One-time outflow.

  • Fields: year, amount (usually negative)
  • expense_kind = "mandatory" | "discretionary"
  • funding = "cash_reserve_first" to let this event break reserve protection first
Income

One-time or bounded extra income stream.

  • Fields: year, amount
  • Optional: end_year, taxable, taxable_fraction
BuyHome

Uses cash for a purchase and, when `price` is supplied, creates/updates a tracked property.

  • Fields: year, down_payment, price
  • Optional: property, mortgage_rate, term_years
SellHome

Converts a named property into net cash proceeds and can optionally reinvest some or all of it.

  • Fields: year, property
  • Optional: liability_names, sale_fee_rate, reinvest_to, reinvest_fraction
NewJob / CareerBreak / Education / Marriage / SpendingShift

Additional event types for job changes, pauses, tuition, informational life events, and baseline-spending regime changes.

ContributionChange

Override one or more per-person contribution amounts starting from a given year. Useful for modelling a mid-scenario step-up or step-down in 401(k) or IRA contributions (e.g. restoring full 401(k) contributions after an early mortgage payoff).

  • Required: year, person (person1 or person2)
  • Optional: annual_401k_contribution — new absolute dollar amount
  • Optional: annual_ira_contribution — new absolute dollar amount
  • Optional: annual_401k_employer_match — new employer match dollar amount (flat mode)
  • Optional: annual_401k_contribution_delta — relative dollar change (+/-) to 401(k) contribution
  • Optional: annual_ira_contribution_delta — relative dollar change (+/-) to IRA contribution
  • Optional: annual_401k_employer_match_delta — relative dollar change (+/-) to employer match
  • Optional: annual_401k_employer_match_mode — "flat" or "percent_of_gross"
  • Optional: annual_401k_employer_match_rate — match rate as decimal (e.g. 0.50 = 50% match)
  • Optional: annual_401k_employer_match_max_percent — max salary % matched (e.g. 0.06)
  • Optional: gross_income — new gross income for percentage-based contributions
  • Optional: gross_income_annual_increase_percent — new gross income increase rate
  • Optional: retirement_contribution_percent — new contribution percentage
  • Optional: retirement_contribution_annual_increase_percent — new escalation rate
  • Optional: retirement_contribution_max_percent — new percentage cap
  • Multiple events for the same person are applied in year order; later years win
Recurring controls

Repeat a compatible event without copying it many times.

  • repeat_every_years
  • repeat_until_year
  • repeat_count
  • chart_first_occurrence_only for decluttering labels
