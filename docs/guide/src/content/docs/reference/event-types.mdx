---
title: Event Types Reference
description: All supported event types and their properties
---

Events are declared in your scenario TOML as `[[events]]` array entries. Each event has a `type` that determines its property schema and how it affects the model.

## Common Properties

All events share these properties:

| Property | Type | Required | Default | Description |
|---|---|---|---|---|
| `enabled` | bool | Yes | — | Set to `false` to disable without deleting |
| `type` | string | Yes | — | Determines the event behavior |
| `label` | string | No | — | Display label on chart annotations |
| `year` | int | Varies | — | The year the event occurs (required for most types) |

## Event Types
### Retire

Permanent from the specified year. Stops earned income for that person.

```toml
[[events]]
enabled = true
type = "Retire"
person = "person1"      # Required: "person1" or "person2"
year = 2038
```

Retirement and SocialSecurity events are auto-synthesized from per-person settings. Manual entries override the auto-synthesis.

### SocialSecurity

Permanent from the specified year. Adds monthly Social Security income.

```toml
[[events]]
enabled = true
type = "SocialSecurity"
person = "person1"         # Required
year = 2035
monthly_benefit = 2400     # Required: monthly SS payment
```
### Expense

One-time cash outflow. Can be recurring.

```toml
[[events]]
enabled = true
type = "Expense"
label = "Kitchen remodel"
year = 2028
amount = -45000                    # Negative = cash outflow
funding = "cash_reserve_first"     # Optional: let expense tap cash reserve
repeat_every_years = 5             # Optional: recurrence interval
repeat_until_year = 2048           # Optional: last recurrence year
repeat_count = 3                   # Alternative to repeat_until_year
chart_first_occurrence_only = true # Optional: show only first annotation
```
### Income

Adds income for a period (or a single year).

```toml
[[events]]
enabled = true
type = "Income"
label = "Side consulting"
year = 2027
amount = 15000                     # Positive = cash inflow
end_year = 2029                    # Optional: last year of income
```
### BuyHome

Records a property purchase with optional mortgage.

```toml
[[events]]
enabled = true
type = "BuyHome"
label = "Beach house"
year = 2032
down_payment = -80000              # Cash outflow at purchase
price = 450000                     # Total purchase price
property = "Beach House"           # Optional: name for tracking
mortgage_rate = 0.065              # Optional: mortgage annual rate
term_years = 30                    # Optional: mortgage term
```
### SellHome

Converts a named property to cash, net of fees and mortgage payoff.

```toml
[[events]]
enabled = true
type = "SellHome"
label = "Sell current home"
year = 2032
property = "Primary Residence"     # Required: must match BuyHome or property name
sale_fee_rate = 0.06               # Optional: realtor fees (default 6%)
liability_names = ["Home Mortgage"] # Optional: liabilities to pay off at sale
reinvest_to = "taxable"            # Optional: bucket to deposit proceeds into
reinvest_fraction = 0.5            # Optional: fraction to reinvest (rest stays cash)
```
### NewJob

Replaces a person's income from the specified year.

```toml
[[events]]
enabled = true
type = "NewJob"
person = "person2"
year = 2026
annual_income = 85000
```
### CareerBreak

Zeroes a person's earned income for a period.

```toml
[[events]]
enabled = true
type = "CareerBreak"
person = "person1"
start_year = 2030                  # First year of break
end_year = 2031                    # Last year of break (inclusive)
```
### Education

Subtracts annual cost for a period.

```toml
[[events]]
enabled = true
type = "Education"
person = "person1"
start_year = 2027
end_year = 2030
annual_cost = 15000
```
### Marriage

Currently informational. Future: tax filing status change.

```toml
[[events]]
enabled = true
type = "Marriage"
year = 2027
```
### SpendingShift

Changes baseline retirement spending from a specified year.

```toml
[[events]]
enabled = true
type = "SpendingShift"
label = "Downsize lifestyle"
year = 2040
mode = "replace"                   # Only mode currently supported
retirement_annual = 65000          # New retirement spending level
survivor_percent_of_retirement = 0.70  # New survivor %
end_year = 2050                    # Optional: revert after this year
```
## Recurring Event Fields

Events that support recurrence (currently `Expense` and `Income`):

| Field | What It Does |
|---|---|
| `repeat_every_years` | How many years between each repeat |
| `repeat_until_year` | The last year the event occurs (mutually exclusive with `repeat_count`) |
| `repeat_count` | Total number of occurrences (including the first) |
