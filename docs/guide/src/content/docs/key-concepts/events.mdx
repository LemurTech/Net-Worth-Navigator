title: Events &amp; the Event System
description: Model life events that change your financial trajectory
---

Events are how you tell the model about things that will happen in the future — a planned expense, a career change, a home purchase, or retirement. They're declared in your scenario TOML under `[[events]]`.

## How Events WorkSection titled "How Events Work"

Each event has:

- A **type** that determines how it affects the model
- A **year** (or start/end years) specifying when it occurs
- An **amount** or other type-specific parameters
- An **enabled** flag — disable events without deleting them

Events can be singular (one year) or span multiple years. They can apply to one person or the household.

### RetireSection titled "Retire"
```toml
[[events]]
enabled = true
type = "Retire"
person = "person1"
year = 2038
```

Sets that person's earned income to zero from the specified year onward. Also triggers retirement-phase spending and withdrawal policies.

### SocialSecuritySection titled "SocialSecurity"
```toml
[[events]]
enabled = true
type = "SocialSecurity"
person = "person1"
year = 2035
monthly_benefit = 2400
```

Adds monthly income from the specified year onward. Retirement and SS events are now **auto-synthesized** from the per-person settings in your TOML — you usually don't need to write them manually. If you do add them, your metadata overrides (like label or enabled flag) are preserved.

### ExpenseSection titled "Expense"
```toml
[[events]]
enabled = true
type = "Expense"
label = "Kitchen remodel"
year = 2028
amount = -45000
```

A one-time cash outflow. Negative amounts reduce liquid assets. Optional `funding = "cash_reserve_first"` lets that expense draw from cash below target before touching retirement buckets.

### Recurring EventsSection titled "Recurring Events"

Events can repeat using three fields:

```toml
[[events]]
enabled = true
type = "Expense"
label = "Annual travel"
year = 2027
amount = -8000
repeat_every_years = 1
repeat_until_year = 2045
```

Field
What It Does

`repeat_every_years`
How many years between repeats

`repeat_until_year`
The last year the event repeats

`repeat_count`
Alternative: repeat exactly N times (ignored if `repeat_until_year` is set)

For readability on the chart, set `chart_first_occurrence_only = true` to show annotations only for the first occurrence.

### IncomeSection titled "Income"
```toml
[[events]]
enabled = true
type = "Income"
label = "Side consulting"
year = 2027
amount = 15000
end_year = 2029
```

Adds annual income for the specified period. Positive amounts increase cash flow.

### BuyHome / SellHomeSection titled "BuyHome / SellHome"
```toml
[[events]]
enabled = true
type = "BuyHome"
label = "Move to coast"
year = 2032
down_payment = -80000
price = 450000

[[events]]
enabled = true
type = "SellHome"
label = "Sell current home"
year = 2032
property = "Primary Residence"
sale_fee_rate = 0.06
```

`BuyHome` records a new property and subtracts the down payment. `SellHome` converts a named property to cash, net of sale fees and any linked mortgage payoff. Reinvestment into a specific bucket is supported via `reinvest_to`.

### NewJob / CareerBreakSection titled "NewJob / CareerBreak"
```toml
[[events]]
enabled = true
type = "NewJob"
person = "person2"
year = 2026
annual_income = 85000

[[events]]
enabled = true
type = "CareerBreak"
person = "person1"
start_year = 2030
end_year = 2031
```

`NewJob` replaces a person's income from that year. `CareerBreak` zeroes income for the specified period.

### SpendingShiftSection titled "SpendingShift"
```toml
[[events]]
enabled = true
type = "SpendingShift"
label = "Downsize lifestyle"
year = 2040
mode = "replace"
retirement_annual = 65000
survivor_percent_of_retirement = 0.70
```

Changes the baseline retirement spending from the specified year. Currently supports `mode = "replace"` which replaces the scenario's spending parameters.

## Organizing Events in the Gantt TimelineSection titled "Organizing Events in the Gantt Timeline"

The [Gantt tab](/Net-Worth-Navigator/key-concepts/projection/) shows all enabled events across the projection timeline. Milestone events (retirement, SS start, singular expenses) appear as single markers. Span events (career breaks, education periods) appear as horizontal bars. Survivor periods are shown as a shaded band.

## Toggling EventsSection titled "Toggling Events"

Every event can be toggled on and off by changing `enabled = true` to `enabled = false`. No need to delete entries — just disable them. This is useful for comparing the impact of a specific event (e.g., "what if we don't buy the vacation home?") without creating a separate scenario.
